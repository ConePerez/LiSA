Run algorithms automatically

This tool allows you to generate random instances of scheduling problems using Talliard's generator and run various algorithms with this problem. Currently supported are O||Cmax and O||SumCi.

At first you have to create an input file. (e.g. ~/test/myproblem.autoalg) Then switch to the lisa binary directory (e.g ~/LiSA/bin) and call auto_alg giving the input file as parameter. The file will then be processed and for each problem instance an output file, named [inputfilename].[number].xml will be created, containing the problem instance and all solutions that were generated by the used algorithms.

Please note that the first algorithm to be called has to be constructive because it can not receive an input solution. For all further algorithms you can provide schedules computed by ealier algorithms as input. They can therefore either be constructive or iterative.

The generated output files can be opened with LiSA. They do not only contain the problemtype, instance and all generated solutions but also all information neccesary to reproduce the results.

All output generated by the algorithms will appear on standard output. You may want to log them for further investigation.

The first entry is the wanted <PROBLEMTYPE>. Then follows a <CONTROLPARAMETERS> entry with parameters for auto_alg. Then <CONTROLPARAMETERS> for the algorithms which should be called follow, each with an additional AUTOALG_EXECUTABLE entry. Further parameters that can be given to an algorithm are AUTOALG_START_FROM and AUTOALG_TIMELIMIT.

Parameters for auto_alg

M,N
The number of machines and jobs in the problem instances to be generated.
TIMESEED,MACHSEED
The timeseed and machineseed for the random generation of instances.
MINPT,MAXPT
Upper and lower bound for the random generation of processingtimes.
NUMBERPROBLEMS
How many problems should be generated ?
NUMBERALGORITHMS
How many algorithms should be called for each generated instance ? This number has to match the number of following <CONTROLPARAMETERS> entrys.

Additional parameters for an algorithm

AUTOALG_EXECUTABLE
Required! The executable which needs to be started to run this algorithm.
AUTOALG_TIMELIMIT
The algorithm will be inerrupted with SIGINT when its timelimit is reached. This is real time in seconds, not process time. The default value is 0 meaning no limit. This should only be used with algorithms which write output when catching SIGINT.
AUTOALG_START_FROM
The output of which algorithms should be given as input to this algorithm. The syntax is string AUTOALG_START_FROM {2,3}, without any whitespaces in the {2,3} part. The numbers given must be lower then the number of the current algorithm. {0} indicates no input to the algorithm. The default value for algorithm i is {i-1}.
