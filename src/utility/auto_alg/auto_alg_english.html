Run algorithms automatically

This tool allows you to generate random instances of scheduling problems using Talliard's generator and run various algorithms with this problem. Currently supported are O||Cmax and O||SumCi.

Please note that the first algorithm to be called has to be constructive because it can not receive an input solution. All further algorithms will get the solution from the previous algorithm as input. They can therefore either be constructive or iterative.

The generated output files can be opened with LiSA. They do not only contain the problemtype, instance and all generated solutions but also all information neccesary to reproduce the results.

All output generated by the algorithms will appear on standard output. You may want to log them for further investigation.

Howto

At first you have to create an input file. (e.g. ~/test/myproblem.autoalg) Then switch to the lisa binary directory (e.g ~/LiSA/bin) and call auto_alg giving the input file as parameter. The file will then be processed and for each problem instance an output file, named [inputfilename].[number].lsa will be created, containing the problem instance and all solutions that were generated by the used algorithms.

An input file

<PROBLEMTYPE>
 Lisa_ProblemType= { O / / Cmax }
</PROBLEMTYPE>

<CONTROLPARAMETERS>
 long M 10
 long N 10
 long MINPT 1
 long MAXPT 20
 long TIMESEED 89327548
 long MACHSEED 23485327
 long NUMBERPROBLEMS 2
 long NUMBERALGORITHMS 2
</CONTROLPARAMETERS>

<CONTROLPARAMETERS>
 string EXECUTABLE dispatch
 string ACTIVE TRUE
 string RULE SPT
</CONTROLPARAMETERS>

<CONTROLPARAMETERS>
 string EXECUTABLE nb_iter
 long STEPS 5000
 long NUMB_STUCKS 10000
 long PROB 12
 long MAX_STUCK 30
 long TABULENGTH 40
 long NUMB_NGHB 40
 double ABORT_BOUND 0
 string NGBH CR_SHIFT
 string METHOD SimulatedAnnealing
 string TYPE RAND
</CONTROLPARAMETERS>
The first entry is the wanted <PROBLEMTYPE>. Then follows a <CONTROLPARAMETERS> entry with parameters for auto_alg. Then <CONTROLPARAMETERS> for the algorithms which should be called follow, each with an additional EXECUTABLE entry which names the executable for this algorithm.

Parameters

M,N
The number of machines and jobs in the problem instances to be generated.
TIMESEED,MACHSEED
The timeseed and machineseed for the random generation of instances.
MINPT,MAXPT
Upper and lower bound for the random generation of processingtimes.
NUMBERPROBLEMS
How many problems should be generated ?
NUMBERALGORITHMS
How many algorithms should be called for each generated instance ? This number has to match the number of following <CONTROLPARAMETERS> entrys.
