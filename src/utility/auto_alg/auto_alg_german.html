Algorithmen automatisch ausführen

Dieses Programm ermöglicht es automatisch mittels Talliard's Zufallsgenerator Instanzen von Schedulingproblemen zu erzeugen und für diese bestimmte Algorithmen aufzurufen. Zur Zeit werden O||Cmax und O||SumCi unterstützt.

Als erstes muss eine Eingabedatei erzeugt werden. (z.B. ~/test/myproblem.autoalg) Anschliessend wechseln sie ins LiSA binary Verzeichnis (z.B. ~/LiSA/bin) und rufen auto_alg mit der Eingabedatei als Parameter auf. Diese Datei wird dann abgearbeitet und für jede erzeuge Instanz wird eine Ausgabedatei mit dem Namen [inputfilename].[number].lsa erzeugt. Diese enthält die erzeugte Probleminstanz sowie alle erzeugten Lösungen.

Bitte beachten sie das der erste aufzurufende Algorithmus konstruktiv sein muss, da keine bereits bestehende Lösung übergeben werden kann. Alle weiteren Algorithmen erhalten die Lösung des vergehenden Algorithmus als Eingabe und können daher konstruktiv oder iterativ sein.

Die erzeugten Ausgabedateien können mit LiSA geöffnet werden. Sie enthalten nicht nur den Problemtyp, die erzeugte Instanz und alle berechneten Lösungen, sondern auch alle zur Reproduktion der Resultate nötigen Informationen.

Alle von den Algorithmen erzeugten Meldungen erscheinen in der Standardausgabe.

Eine Eingabedatei

<PROBLEMTYPE>
 Lisa_ProblemType= { O / / Cmax }
</PROBLEMTYPE>

<CONTROLPARAMETERS>
 long M 10
 long N 10
 long MINPT 1
 long MAXPT 20
 long TIMESEED 89327548
 long MACHSEED 23485327
 long NUMBERPROBLEMS 2
 long NUMBERALGORITHMS 2
</CONTROLPARAMETERS>

<CONTROLPARAMETERS>
 string EXECUTABLE dispatch
 string ACTIVE TRUE
 string RULE SPT
</CONTROLPARAMETERS>

<CONTROLPARAMETERS>
 string EXECUTABLE nb_iter
 long STEPS 5000
 long NUMB_STUCKS 10000
 long PROB 12
 long MAX_STUCK 30
 long TABULENGTH 40
 long NUMB_NGHB 40
 double ABORT_BOUND 0
 string NGBH CR_SHIFT
 string METHOD SimulatedAnnealing
 string TYPE RAND
</CONTROLPARAMETERS>
Der erste Eintrag enthält den gewünschten <PROBLEMTYPE>. Dann folgt ein <CONTROLPARAMETERS> Eintrag mit Parametern für auto_alg. Anschliessend folgen weitere <CONTROLPARAMETERS> Einträge für die aufzurufenden Algorithmen. Diese enthalten zusätzlich zu den Parametern für die Algorithmen einen EXECUTABLE Eintrag mit dem Namen des aufzurufenden Algorithmus.

Parameter

M,N
Anzahl der Maschinen und Aufträge in den zu erzeugenden Probleminstanzen.
TIMESEED,MACHSEED
Initialisierungswerte für den Zufallsgenerator.
MINPT,MAXPT
Untere und obere Grenze für die zu erzeugenden Bearbeitungszeiten.
NUMBERPROBLEMS
Wie viele Probleme sollen erzeugt werden ?
NUMBERALGORITHMS
Wie viele Algorithmen sollen aufgerufen werden ? Diese Zahl muss mit der Anzahl der folgenden <CONTROLPARAMETERS> Einträge übereinstimmen.
