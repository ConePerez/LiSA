partition Algorithmus

Benutzen Sie diesen Algorithmus, um Probleme des Typs P2 // Cmax zu loesen.
Er basiert auf einem pseudopolinomialen Algorithmus zur Lösung des Problems PARTITION, der das Prinzip der dynamischen Programmierung basiert. Er wurde hier 2001 implementiert von Claudia Isensee.

So kommen Sie hierher:

Stellen Sie sicher, daß ein Problemtyp (P2 // Cmax) und Problemparameter definiert sind. Wählen Sie dann den Menüpunkt Algorithmen|Exakte Verfahren|partition-module.

Parameter

Das Problem benötigt nur ein pi pro Job. Zur Zeit wird aber noch eine zweispaltige Matrix angeboten. Es reicht aus, die erste Spalte dieser Matrix zu füllen.

Probleme:

  • Zur Zeit wird bei P2 // Cmax noch nicht partition angeboten, deshalb tarnen sie Ihr Problem als O2 // Cmax. Nur die Bearbeitungszeiten in in der ersten Spalte von P sind für den Algorithmus von Interesse.
  • Die Anzeige ist auch nur über einen Umweg möglich. Sie müssen ~/.lisa/proc/algo_out.lsa öffnen, um die Lösung anzuzeigen.

Algorithmus

Input

  • Matrix P
  • Anzahl der Jobs n

Output

  • Matrix SIJ
  • Schedule LR
  • Cmax

Algorithmus

  1. berechne die Summe der Bearbeitungszeiten aller Jobs sump
  2. generiere eine Tabelle T wie folgt
    Spalten repräsentieren die verschieden-großen Partitionen von 0 bis sump/2
    Zeilen repräsentieren die verschiedenen Bearbeitungszeiten pi
    0 1 2 ... (sump/2)-1 sump/2
    p1 False False False ... False False
    p2 False False False ... False False
    ... ... ... ... ... ... ...
    pn False False False ... False False
  3. setze T["pi"][0]:=True für alle 1<=i<=n
    setze T["p1"][p1]:=True
  4. ändere T Zeile für Zeile nach den folgenden Regeln, beginnend mit p2
    T["pi"][j]:= True IF
    • pi==j                                  pi ist alleine eine Partition der Größe j
    • T["pi-1"][j]==True            die p1 bis pi-1 können eine Partition der Größe j bilden
    • T["pi-1"][j-pi]==True            die p1 bis pi-1 können eine Partition der Größe j-pi bilden
  5. finde das letzte "True" in Zeile pn bei T["pn"][j]
    j ist die Größe der ersten Partition
  6. finde die Elemente der ersten Partition mit folgenden Schritten während j>0
    1. gehe "hoch" während T["pi"][j] noch "True" ist
    2. nimm dieses pi in die Liste auf
    3. gehe nach "links" für pi Schritte
    4. wiederhole diese Schritte
  7. nimm den Rest der Jobs in Partition 1
  8. bearbeite alle Jobs aus Partition 1 auf M1 in beliebiger Reihenfolge
    bearbeite alle Jobs aus Partition 2 auf M2 in beliebiger Reihenfolge
  9. bilde SIJ und LR entsprechend dieses Schedules
  10. Cmax ist die Summe der Bearbeitungszeiten der Jobs der zweiten Partition
