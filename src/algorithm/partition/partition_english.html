partition Algorithm

Use this algorithm to solve problems of the type P2 // Cmax.
It is based on a pseudopolynomial algorithm for the PARTITION problem, wich uses the principle of dynamic programming. This implementation was done in 2001 bei Claudia Isensee.

This is, how You get here:

Make sure, that You have specified Problem Type (P2 // Cmax) and Parameters. Choose the menu item Algorithms|Exact Algorithms|partition-module.

Parameters

The problem requires only one pi per job. Right now there is still offered a two column matrix. It is sufficient to fill in the first column of the matrix.

Problems:

  • A problem of the type P2 // Cmax does not yet offer partition as an algorithm, therefore You have to camouflage you problem as O2 // Cmax. Only the processing times in the first column of P are of interest for the algorithm.
  • Displaying the solution is only possible through opening ~/.lisa/proc/algo_out.lsa, for now.

Algorithm

Input

  • matrix P
  • number of jobs n

Output

  • matrix SIJ
  • schedule LR
  • Cmax

Algorithm

  1. calculate the sum of the processing times of all jobs sump
  2. generate a table T as seen below
    columns represent the different sized partitions from 0 to sump/2
    rows represent the different values of the pi
    0 1 2 ... (sump/2)-1 sump/2
    p1 False False False ... False False
    p2 False False False ... False False
    ... ... ... ... ... ... ...
    pn False False False ... False False
  3. set T["pi"][0]:=True for all 1<=i<=n
    set T["p1"][p1]:=True
  4. change T row by row with the following rules, starting with p2
    T["pi"][j]:= True IF
    • pi==j                                  pi is a partition on its own of size j
    • T["pi-1"][j]==True            the p1 to pi-1 can build a partition of size j
    • T["pi-1"][j-pi]==True            the p1 to pi-1 can build a partition of size j-pi
  5. find the last "True" in line pn at T["pn"][j]
    j is the size of the first partition
  6. trace the elements of the first partition with the following steps while j>0
    1. go "up" while T["pi"][j] is still "True"
    2. take this pi into the list
    3. go "left" for pi steps
    4. repeat
  7. put the rest of the jobs into the other partition
  8. process all jobs of the first partition on M1 in any order
    process all jobs of the second partition on M2 in any order
  9. build SIJ and LR according to this schedule
  10. Cmax is the sum of the processing times of the jobs in the second partition
