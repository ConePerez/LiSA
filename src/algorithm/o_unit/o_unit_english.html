o_unit algorithm

Use this algorithm to solve an open shop problem with Cmax,SumCi objective heuristically. This algorithm is for problems with unit processing times exact. For problems with more jobs n than machines m, the sequence LR[n,m,x] is divided into latin squares of order m and one latin rectangle B := LR[m+b,m, x].

What you have to do:

Make sure that both a problem type and all parameters are defined. Choose Exact Algorithms|o_unit or Heuristic Algorithms|o_unit from the menu Algorithms and after editing the options press OK.

Options:

  • Parameters

    • LR parameter
      The LR parameter defines the postion where the latin square B is inserted into the sequence.
    • Random seed
      The random seed is used to initiate the random function in the algorithm
    • Presorting rule
      Choose the presorting rule - how to order the jobs
      • RAND
        no change of the job order
      • LPT
        longest processing times(sum) first
      • SPT
        shortest processing times(sum) first
      • MINPTMIN
        shortest processing time first
      • MINPTMAX
        shortest processing time last
      • MAXPTMIN
        longest processing time first
      • MAXPTMAX
        longest processing time last
      • EDD
        earliest duedate first

Problems:

  • The exact algorithm is laid out for complete operation sets. The result of this algorithm is for some O/p_ij=1/Cmax,SumCi problems with incomplete operation set not optimal.
